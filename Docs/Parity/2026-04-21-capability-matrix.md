# Chatbook Server Capability Matrix

Audit date: 2026-04-21

Source spec: `Docs/superpowers/specs/2026-04-21-chatbook-server-capability-parity-audit-design.md`

## Legend

- Action coverage values: `None`, `Partial`, `Substantial`, `Full`
- Primary UI mode values: `Separated Local/Server`, `Remote-only`, `Local-only`, `Optional mixed later`
- Client obligation values: `Full CRUD`, `Discover / Configure / Trigger / Observe`, `Discover / Trigger / Observe`, `Observe-only`, `Unavailable offline with explicit fallback`

## Top Priority Rows

- `Collections: Reading List / Read-it-later` (81): Directly matches the standalone-first reading-collection mandate; local saved-state persistence, server save/remove compatibility, and the aggregate `All Media` saved view are now landed with contract-honest UI enforcement, while any future per-media-type server saved view requires a server contract extension rather than more Chatbook-only shaping.
- `Watchlists` (81): The source-aware subscriptions shell, remote watchlist source CRUD/restore, server jobs/runs/alert-rule administration, server reminders/feed tabs, and local notifications inbox are now landed; groups, richer structured control-plane UX, and sync remain deferred.
- `Writing Suite` (78): The user explicitly wants serious standalone writing support, and Chatbook still lacks a dedicated project and manuscript hierarchy.
- `Media / Reading / Ingestion Sources` (78): Reading progress, ingestion provenance, and media CRUD underpin the broader read-it-later and research parity story.
- `Chat` (78): Core chat credibility depends on preserving strong local operation while closing remote conversation contract gaps.
- `Research Sessions / Runs` (75): The source-separated Research Sessions surface is now landed with local run storage/control, server research-runs client support, selected-run live server event observation, mounted bundle/artifact/checkpoint/event-log depth, and a dedicated Research TUI destination; the real remaining gap is a local execution engine plus further UX polish.
- `Local MCP Runtime` (73): Local MCP operations are a stated priority and unlock serious standalone automation without forcing server dependence.
- `Characters / Personas / CCP` (73): Persona and character flows are already central to chat usage, and shared identifiers still need to align across local and server modes.
- `Notes / Workspaces` (73): Workspace-aware local/server note parity remains a major daily-use surface with existing seams worth finishing.
- `Client Notifications` (71): Persisted local queue storage, filtered queue reads, dispatch, policy-gated delivery preferences, and inbox read/dismiss behavior are now landed for watchlists and subscriptions actions; broader producers remain separate, and server feed surfaces are now exposed separately as server-owned state.
- `Cross-cutting Runtime Policy` (70): The tranche-0 foundation is now landed, and the row remains important as an extension surface that reduces parity regressions across many rows.

Chatbook has one authoritative capability seam, runtime normalization at backend switch, screen entry, and pre-query, plus a search-panel status affordance that makes the boundary explicit: local `Read-it-later` remains per-media-type, while server `Read-it-later` is aggregate-only in `All Media`. Remaining work is now mostly UI and parity cleanup rather than missing route-family support on the active hub-management surface; only deprecated alias paths remain intentionally uncovered.

Status update: `Research Sessions / Runs` now has mounted detail depth in addition to the original source-separated tranche slice. Chatbook includes typed research-runs schemas and client methods, typed server SSE event streaming, local SQLite-backed research run and artifact storage, local/server scope routing under runtime policy, a source-switched Research screen with selected-run event watching, explicit bundle and artifact loading, checkpoint approve controls, a persistent event log, app navigation wiring, and focused tests. Remaining gaps are a real local research execution engine and further UX polish rather than missing mounted observe/control seams.

Status update: `Prompts / Chatbooks` now has its first contract seam and source-routed prompt CRUD in the existing CCP prompt editor. Chatbook includes typed server prompt CRUD methods, typed server prompt collection create/list/detail/update methods, local prompt collection create/list/detail/update persistence behind the same scope service, typed server chatbook export/import/job list/status/cancel/remove/download/continue/cleanup methods, a server chatbook service that preserves existing wizard dict contracts, writes downloaded exports locally, exposes explicit continuation routing, and exposes expired-export cleanup, live remote server job browsing plus cancel/download/remove controls in the export management window, a normalized local/server prompt scope service, source-aware server prompt usage/version/restore routing, app-level prompt scope wiring, CCP prompt list/load/create/update/delete routing through the active runtime source, and mounted CCP controls for prompt usage, server version listing, and server version restore. Remaining gaps are prompt workflows, mounted chatbook cleanup/continuation controls, and deeper import/export identity alignment.

Status update: `Chat` now has backend-only source-aware conversation and chat-loop seams. Chatbook wires local conversation metadata/history/delete, server conversation list/detail/update/tree operations, server chat-loop start/event/approve/reject/cancel operations, server conversation share-link operations, public share-token resolution, server RAG-context persistence/readback, messages-with-context, and conversation citation retrieval behind source-scoped runtime-policy actions, preserving local scope filtering without changing the mounted chat UI. Remaining gaps are mounted remote launch/message handoff, remote delete pending a server conversation-delete contract, attachment/export identity, and explicit local/server conversation browsing UX.

Status update: `Characters / Personas / CCP` now has a deeper backend-only server contract seam plus a local adapter for the same centralized CCP paths. Chatbook includes typed server character/persona catalog and exemplar methods, server greeting/preset helpers, server character chat session CRUD/settings methods, server character chat message CRUD/search methods, server character memory CRUD/archive/extract methods, local character catalog CRUD/restore, local character chat session/message/settings/memory CRUD, local world-book CRUD/entry CRUD/session attachment/import/export behind the same scope service, scope-service runtime-policy enforcement, and app-level local/server service wiring. Remaining gaps are mounted local/server CCP UX, local memory extraction, shared identifiers, character import/export identity, and any future server-side world-book contract.
