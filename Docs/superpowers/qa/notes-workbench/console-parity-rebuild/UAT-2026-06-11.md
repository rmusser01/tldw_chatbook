## Findings and fixes (all fixed in this pass)

1. **Sort-order toggle invisible in the running app.** The global
   `Select { width: 100%; }` rule (app stylesheet) overrides the pane's
   `DEFAULT_CSS`, inflating the sort select and pushing the button outside
   the clipped row. Fixed with scoped `#notes-sort-row` rules. The bare
   harness never showed this — app-stylesheet rules beat `DEFAULT_CSS`
   regardless of specificity.
2. **Status row grammar** — "1 notes" → "1 note".
3. **Notes list stale after sync.** A sync run that imports notes did not
   refresh the Notes-mode list or status row. `perform_sync` now calls the
   screen's `refresh_current_scope()` on success.
4. **CRITICAL: Delete crashed the whole app.** `handle_delete_button`
   awaited `push_screen_wait(dialog)` from the message pump;
   Textual raises `NoActiveWorker` and the app exits ("Session ended").
   Unit tests never caught it because they mock `push_screen_wait`.
   Fixed by dispatching the delete flow via `run_worker(..., exclusive=True)`.
   The same latent crash existed on `#workspace-add-source-button`
   (media picker await) and was fixed the same way.
5. **Duplicated list rows after dialog dismissal.** Dismissing a modal
   resumes the screen, whose resume-refresh raced the delete worker's
   refresh; interleaved `ListView` appends duplicated every row. Fixed by
   serializing `_populate_scope_list_if_available` with an `asyncio.Lock`.
   (A theoretical stale-order window remains if a resume-refresh's data
   fetch loses a race with a concurrent mutation; not observed.)

## Verdict

All exercised flows pass after fixes. The two crash-class bugs (4) and the
duplication bug (5) predate or were masked from the unit-test suite because
dialog waiting is mocked there; live UAT was the only way to catch them.
